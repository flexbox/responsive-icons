Use HTML character entities in CSS for responsive design

If you are familiar with HTML markup you know character entities. They are commonly used for blockquote, shapes, arrows, separator, ...

For example : &star; &hearts; &rarr;

It’s not necessary to use these codes if you’re encoding pages in UTF-8, but this markup can save your life especially in the cross-encoding nightmare. If you’re not using UTF-8, adjacent characters are causing issues, and it’s difficult to remember specific code for characters. Similarly, you may be using ANSI or another encoding method for CSS.

I use fileformat.info to discover amazing symbols usefull in webdesign. Feel free to to try my webapp for character entity .

Character entities in CSS

If you have already use a webfont icon library, you know how to add content with pseudo element' To add UTF-8 character, you need to find its number and convert that to hexadecimal. For star, it’s 2606. This can be added within a CSS file using a preceding backslash. 

.star-o:before { 
 content: "\2606"; 
}
